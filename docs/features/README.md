# Feature Map

## Existing Features

- Read Jira and Azure DevOps sprint data through MCP/API/Playwright fallback.
- Filter work by scope (`all`, `mine`, selected keys).
- Detect repo technology stacks.
- Build architecture documentation baseline when mapping is weak.
- Plan item-to-repo delivery routes with confidence levels.
- Run dev/build, lint, unit tests, E2E tests, and security review.
- Capture test evidence and PR body context.
- Resume runs through `.sendsprint/runs/`.
- Create GitHub/Azure DevOps PRs.
- Run optional LLM code generation with budget controls.
- Run optional deploy/status callbacks.
- Serve a local FastAPI API and dashboard run screen.

## Active Roadmap Features

- PyPI trusted publishing restoration.
- `sendsprint doctor`.
- Full dry-run plan artifacts.
- Per-task worktree isolation metadata and cleanup.
- Portable evidence bundles.
- GitHub Issues as a native tracker.
- Central autonomy policy.
- Ralph Wiggum and Codex Goal loop semantics.
- Stack validation templates for Angular, React, Vue.js, Node.js, Python, PHP,
  .NET, mobile, and monorepo projects.
- Local dashboard fed by run report/evidence bundle.
- Transcript-to-task extraction for meeting notes.
- Sprint Autopilot demo.
- Executive reports.
- Multi-agent control plane.
